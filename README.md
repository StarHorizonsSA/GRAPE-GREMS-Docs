# <span id="_1doq3c4nks3b"></span> **Glossary**

**GREMS:** GRAPE Rewards & Emissions Management System; the end-to-end rewards calculation and payout documentation framework.

**GRAPE MEMBERS:** Canonical identity mapping (wallets/IDs) used to ensure rewards map to the correct member identifiers.

**Budget Pools per Category:** Approved budgets that constrain category-level distribution totals.

**SCORE: PERFORM & GUBA EMIT:** Monthly calculation layer for Performance (GRAPE / USDC / SOL).

**REF: PERFORM:** Payout-ready GRAPE totals per member (Performance).

**REF: GUBA:** Payout-ready USDC + SOL totals per member (Performance).

**SCORE: MERIT EMIT & GUBA MERIT:** Period calculation layer for Merit (GRAPE / USDC).

**REF: MERIT:** Payout-ready GRAPE totals per member (Merit).

**REF: GUBA MERIT:** Payout-ready USDC totals per member (Merit).

**GOV PARTICIPATION:** Governance participation tracking (eligibility, voting, calls, raffles). Standalone pipeline.

**VINE Rewards:** Reward stream used for Governance Participation payouts; not included in Emissions Results artifacts.

**RECURRING PAYMENTS:** Monthly payment list for add-ons (Proposal Architect, Discord Boost) + infra/tool costs; separate proposal.

**Emissions Results Interface:** Rollup/reporting tab used for presentation and utilization checks; not the same as a system-level Emissions Results artifact/proposal.

**No manual overrides:** Control principle: calculations and identity mapping should remain deterministic and auditable; avoid ad-hoc edits.
